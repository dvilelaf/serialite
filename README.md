# ESP32-KVM

Emergency serial console for headless Linux servers.

ESP32-KVM plugs into the server over USB and creates its own local WiFi network. From a laptop or phone, you open a web terminal and use the server serial console even when SSH or networking is broken.

This is not a video KVM: there is no HDMI capture, remote HID keyboard, or virtual media. It is an emergency serial console.

## Quick Start

1. Plug ESP32-KVM into the server USB port.

2. On the server, run the host setup once:

   ```bash
   sudo ./tools/host/setup-linux-serial-console.sh
   ```

3. Join the WiFi network from your laptop or phone:

   ```text
   SSID: KVM
   Password: shown on the ESP32 screen
   ```

4. Open:

   ```text
   http://192.168.4.1
   ```

5. Use the terminal.

The setup script detects the ESP32-KVM serial device, enables the right `serial-getty@<tty>` service, and makes it persistent across reboots. If the server already has a working serial console, the script is harmless to run again.

## Buttons

- `BOOT`: wakes the screen and temporarily reveals the WiFi password.
- `PWR` for 3 seconds: locks or unlocks terminal input.
- `BOOT` for 10 seconds: factory-resets ESP32-KVM configuration.

## Web UI

- The main screen is the terminal.
- The `+` button opens quick actions: useful keys, TTY resize, diagnostics, WiFi rotation, and emergency lock.
- `Emergency lock` blocks web terminal input. To unlock it, hold `PWR` for 3 seconds.
- `Rotate WiFi` generates a new password, shows it on the ESP32 screen, and restarts the AP.

## Operational Security

- WiFi uses WPA2 with a password generated by the device.
- Only one WiFi client is allowed.
- Only one web session is active.
- There is no extra web password: real authentication happens at the Linux console.
- Secrets are not returned over HTTP; they are shown physically on the device screen.

## Development

Verify:

```bash
source /home/david/esp-idf/export.sh
./scripts/verify.sh
```

Build and flash:

```bash
source /home/david/esp-idf/export.sh
ls /dev/serial/by-id/*Espressif*
idf.py -p /dev/serial/by-id/<the-esp32-path> flash
```

If flashing fails because of USB permissions or serial noise, reconnect the board and use a lower baud rate:

```bash
idf.py -p /dev/serial/by-id/<the-esp32-path> -b 115200 flash
```
